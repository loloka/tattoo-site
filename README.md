# Сайт-визитка / галерея работ (PHP + MySQL)

Простой сайт: главная страница с галереей работ + форма «связаться» (уходит
в Telegram) + отдельная админка, где можно самостоятельно добавлять,
редактировать и удалять работы — без доступа к коду и без FTP.

Дизайн свёрстан по макету `mashasite.psd` (чёрно-белый, всё капсом,
двуязычный EN/DE). Шрифт макета — Myriad Pro; если он установлен в системе,
браузер возьмёт его сам, если нет — подключается бесплатный Source Sans 3
(открытый «родственник» Myriad с кириллицей).

Технологии: PHP 8, MySQL, ваниль JS/CSS (без фреймворков и сборки — грузится
как есть на любом обычном хостинге).

## 🚀 Коротко: как запустить у себя (Docker)

Самый простой способ посмотреть сайт живьём — не нужно ставить PHP, MySQL и
Apache вручную. Нужен только **Docker Desktop**.

1. Установи [Docker Desktop](https://www.docker.com/products/docker-desktop/),
   запусти его и дождись, пока иконка кита в трее перестанет анимироваться.
2. Распакуй проект в любую папку.
3. **Настрой файл `.env`** — это делается ОДИН раз перед первым запуском,
   подробно расписано в следующем разделе. Для просто «посмотреть» можно и
   не трогать — сайт поднимется с текстами из макета по умолчанию.
4. Открой терминал в папке проекта и выполни:
   ```
   docker compose up -d --build
   ```
   Первый раз — минуту-две (скачивает образы), дальше секунды.
5. Сайт → **http://localhost:8080**
   Админка → **http://localhost:8080/admin/login.php** (логин `admin`,
   пароль `demo1234` — временный, только для локального теста).

Двойной клик по `start.bat` (Windows) / `start.sh` (Mac/Linux) сделает то же
самое и сам откроет сайт в браузере.

## ⚙️ Настройка `.env` перед запуском (главное для Mary)

Все тексты сайта, имя, телеграм и бот настраиваются в одном файле — **`.env`**
в корне проекта. Открой его любым текстовым редактором (Блокнот подойдёт).

### Как это работает — 3 простых правила

1. **Пусто = текст из макета.** Если оставить строку вида `SITE_NAV_HOME=`
   пустой, сайт сам подставит значение по умолчанию из макета (на англ./нем.).
   Заполняй только то, что хочешь поменять.
2. **Значение — в одну строку.** Перенос строки внутри `.env` не работает.
   Чтобы сделать абзац/перенос внутри текста (например в био) — пиши
   буквально `\n`, на странице он превратится в настоящий перенос.
3. **После правки `.env`** выполни `docker compose up -d` (без `--build`) —
   контейнер пересоздастся с новыми значениями. Просто обновить страницу тут
   недостаточно, потому что `.env` читается при старте контейнера.

### Что где менять

```ini
# --- База данных: для локального Docker менять НЕ нужно ---
DB_DATABASE=art_site
DB_USER=art_user
DB_PASSWORD=art_pass
DB_ROOT_PASSWORD=root_pass

# --- Имя и телеграм владелицы ---
SITE_OWNER_FIRST=Marie          # левое слово большого заголовка
SITE_OWNER_LAST=Holzer          # правое слово большого заголовка
SITE_TELEGRAM=@username         # как показывается в шапке и футере
SITE_TELEGRAM_URL=https://t.me/username   # куда ведёт ссылка

# --- Подзаголовок под именем ---
SITE_SUBTITLE=Tattoo Artworks   # жирная надпись
SITE_SUBTITLE_EYEBROW=          # мелкая серая строка НАД ней (пусто = "Personal Art Collection")

# --- Меню в шапке (пусто = Гл. страница / Галерея / Контакты) ---
SITE_NAV_HOME=
SITE_NAV_GALLERY=
SITE_NAV_CONTACT=

# --- Био слева ("About me" / "Über mich") ---
# Абзацы разделяй через \n. Пусто = текст из макета.
SITE_BIO_LABEL=
SITE_BIO_LABEL_DE=
SITE_BIO=Born in Siberia, currently based in Ufa.\nProfessional tattoo artist since 2011.
SITE_BIO_DE=Geboren in Sibirien, derzeit tätig in Ufa.\nProfessionelle Tätowiererin seit 2011.

# --- Короткая подпись контактов рядом с био ---
SITE_CONTACT_LABEL=
SITE_CONTACT_LABEL_DE=

# --- Галерея: эйбрау, заголовок, текст при пустой галерее ---
GALLERY_EYEBROW=
GALLERY_EYEBROW_DE=
GALLERY_TITLE=
GALLERY_TITLE_DE=
GALLERY_EMPTY=
GALLERY_EMPTY_DE=

# --- Форма "связаться": заголовок и пояснение ---
CONTACT_FORM_TITLE=
CONTACT_FORM_TITLE_DE=
CONTACT_FORM_INTRO=
CONTACT_FORM_INTRO_DE=

# --- Футер: строка про телеграм-канал со скетчами ---
SITE_TG_CHANNEL_NOTE=
SITE_TG_CHANNEL_NOTE_DE=

# --- Telegram-бот для формы "связаться" (см. ниже) ---
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

> Заголовок на сайте — это `SITE_OWNER_FIRST` + `SITE_OWNER_LAST`. По макету
> это **Marie Holzer** (имя центрируется на 1/3 ширины, фамилия — на 2/3).

## 🤖 Как подключить Telegram-бота (форма «связаться»)

Когда посетитель заполняет форму на сайте, заявка приходит тебе в Telegram.
Для этого нужны две вещи в `.env`: **токен бота** и **ID чата**, куда слать.

Бота можно оставить пустым — тогда форма всё равно работает (посетитель
видит «Спасибо, отправлено»), просто уведомление никуда не уходит. Настраивай,
когда захочешь реально получать заявки.

### Шаг 1. Создать бота и получить токен

1. В Telegram найди **@BotFather** и напиши ему.
2. Команда **`/newbot`** → он спросит имя бота (любое) и username (должен
   заканчиваться на `bot`, например `marie_holzer_form_bot`).
3. В ответ BotFather пришлёт **токен** вида
   `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`.
4. Вставь его в `.env`:
   ```ini
   TELEGRAM_BOT_TOKEN=123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

### Шаг 2. Узнать ID чата (куда слать заявки)

1. **Напиши своему новому боту любое сообщение** (просто «привет») — без
   этого бот не имеет права написать тебе первым.
2. Открой в браузере ссылку (подставь свой токен вместо `<ТОКЕН>`):
   ```
   https://api.telegram.org/bot<ТОКЕН>/getUpdates
   ```
3. В ответе найди фрагмент `"chat":{"id":123456789,` — вот эти цифры и есть
   твой ID. Вставь в `.env`:
   ```ini
   TELEGRAM_CHAT_ID=123456789
   ```
   (Если ничего в ответе нет — напиши боту ещё раз и обнови страницу.
   Как альтернатива, свой ID можно узнать у бота **@userinfobot**.)

### Шаг 3. Применить

```
docker compose up -d
```
Теперь заполни форму на сайте — заявка должна прийти боту в Telegram.
Хочешь получать заявки в общий чат/канал — добавь бота туда админом и
используй ID этого чата (у групп он с минусом, например `-1001234567890`).

## Нужно ли перезапускать после изменения файлов?

- **Правишь PHP/CSS/JS/HTML** (`public_html/`, `app/`) — ничего перезапускать
  не надо, сохрани файл и обнови страницу (Ctrl+F5, чтобы сбросить кэш CSS).
- **Правишь `.env`** (тексты/имя/телеграм/бот/пароли БД) — нужно
  `docker compose up -d` (без `--build`).
- **Правишь `Dockerfile`** — нужен `docker compose up -d --build`.
- **Правишь `sql/schema.sql`** — накатывается только при первом создании базы.
  Чтобы применить заново: `docker compose down -v` и подними снова (это сотрёт
  добавленные через админку работы).

## Фото в шапке

В hero-блоке слева стоит серый квадрат-заглушка. Чтобы вместо него было фото —
положи картинку в **`public_html/assets/img/hero.jpg`** (квадратное фото
впишется лучше всего). Появится автоматически, перезапуск не нужен.

## База данных глазами (необязательно)

По умолчанию выключено. Включить:
```
docker compose --profile tools up -d
```
Adminer — **http://localhost:8081** (сервер `db`, логин/пароль/база из `.env`).
phpMyAdmin — **http://localhost:8082** (под root, пароль из `.env` →
`DB_ROOT_PASSWORD`).

## Шпаргалка команд

```
docker compose up -d --build          # запустить (первый раз / после Dockerfile)
docker compose up -d                   # перезапустить после правки .env
docker compose logs web                # логи, если что-то не работает
docker compose down                    # остановить (база останется)
docker compose down -v                 # остановить и стереть базу начисто
```

Адрес также виден в Docker Desktop: вкладка **Containers** → группа с именем
папки проекта (`tattoo-art`) → у сервиса `web` в столбце **Port(s)** ссылка
`8080:80`.

## Разворачивание на обычном shared-хостинге (Beget / Timeweb / cPanel и т.п.)

На таких хостингах нет Docker и `.env` — настройки берутся из
**`app/config.php`** (создаётся из `config.example.php`). Значения те же,
что и в `.env`, просто в формате PHP-констант.

1. **Залейте файлы по FTP** так, чтобы `app/` лежал рядом с `public_html/`,
   на уровень выше (тогда его не открыть из браузера):
   ```
   /home/username/
   ├── app/            ← из этого проекта
   └── public_html/    ← из этого проекта
   ```
2. **Создайте базу MySQL** в панели хостинга — получите имя базы,
   пользователя, пароль, хост (обычно `localhost`).
3. **Импортируйте структуру**: phpMyAdmin → ваша база → «Импорт» →
   файл `sql/schema.sql`.
4. **Настройте `app/config.php`**: скопируйте `config.example.php` в
   `config.php`, впишите данные базы, логин админки и (см. шаг 5) хэш пароля.
5. **Хэш пароля админки**: откройте `https://ваш-сайт.ru/admin/tools/hash.php`,
   введите пароль → скопируйте строку в `ADMIN_PASSWORD_HASH` →
   **удалите файл `admin/tools/hash.php`** с хостинга.
6. **Telegram-бот**: получите токен и chat_id так же, как в разделе про бота
   выше, но впишите их в `config.php` (`TELEGRAM_BOT_TOKEN`,
   `TELEGRAM_CHAT_ID`), а не в `.env`.
7. **Тексты и имя** — там же в `config.php` (`SITE_OWNER_FIRST` и т.д.).
8. **Фото шапки** — `public_html/assets/img/hero.jpg`.
9. **Права на папки** — если загрузка фото в админке падает с ошибкой,
   выставьте `755` на `public_html/uploads/works` и `.../thumbs`.

## Как Mary добавляет работы (без правки кода)

1. Заходит на `/admin/login.php`, вводит логин/пароль.
2. «+ Добавить работу» → загружает фото, вписывает название/автора/описание.
3. «Сохранить» — работа сразу в галерее. Там же можно редактировать, удалять
   и задавать порядок показа (меньше число — раньше в галерее).

## Безопасность — не забыть

- `app/config.php` и `.env` не должны попадать в открытый доступ и в git
  (там пароли от базы и токен бота).
- Удалите `admin/tools/hash.php` после первого использования.
- Папка `uploads/` защищена `.htaccess`, запрещающим выполнение PHP внутри.
- Пароль `demo1234` — только для локального теста; на хостинге ставьте свой
  через генератор хэша (шаг 5).
